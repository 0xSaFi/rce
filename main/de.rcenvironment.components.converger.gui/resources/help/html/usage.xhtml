Converger Component Usage

Synopsis

The Converger Component is used to check if data from a workflow loop converges in comparison to its last run.

Rationale

This component checks if all input data from a workflow loop changed more than a given value from the last run. For this, every input channel is tested if its absolute difference from the current run and the last run is smaller than a given value epsilon. This is the absolute convergence criteria. Also there is a test, if it converges relative, so the absolute difference is divided by the old input value. In formula, you have:Let x_old be the values of the last run and x the values of the current run. Also let eps_a be the given absolute convergence criteria and eps_r be the relative convergence criteria.

The component finishes if

|x_old – x| < eps_a

or

|x_old – x| / x < eps_r

is true.

Also, if because of some reason these criteria are never true, there is a parameter to stop the component after a maximum number of iterations is reached.

This check is made for every input variable and the component converges only if all input variables are smaller than eps_a respectively eps_r.

After this check, the component just gives the current values (x) to the corresponding output channel.

For the first run, it is possible to give every variable a start value which will be used for convergence testing. If you don’t use the start value, all values from the very first run won’t be tested but saved for testing the second run.

Usage

The Converger Component has three configuration views. First, there is the convergence criteria view, where you can set the values of the eps parameter (absolute and relative separately). You can also define the number of iterations (k), which should be considered. If k > 1 not only the current and previous value, but the minimum and maximum from the set of current plus previous k values. So k = 1 would mean only current and previous, k = 2 current and two previous, etc. Finally, you can limit the number of convergence checks.

With the second view, you can create the inputs and outputs of the Converger component. Note that you only have to create every variable once because it will create both an input and two outputs endpoints (an additional one for the final converged variable values) with the same type for this variable at the same time. In the "Add" dialog you can also decide whether to use a start value for the variable or not.

When you are finished adding variables, you can connect them in the connection editor. As for the output, the component has three more of them. These are boolean values which can be used to tell other components, if convergence (absolute and relative combined as well as separated) is reached.

If your Converger component is part of a nested loop, you need to check the checkbox in the "Nested Loop" tab. An additional input "Outer loop done" will be created. It tells the Converger component if the outer loop is done (converged, optimized, etc.). Thus, the input must be connected with the outer loop. The outer loop must send a "true" value once it is done. E.g., if the outer loop is controlled by a Converger or Optimizer component, connect their "Outer loop done" output to the Converger's "Outer loop done" input.
