Evaluation Memory Component Usage

Synopsis

The Evaluation Memory component stores results of loop runs and re-uses them in future runs.

Rationale

The Evaluation Memory component is used within loops. It can speed up loops by reusing results of past loop runs. Usually, the Evaluation Memory component is used before/after the loop driver component (e.g., DOE, Optimizer). It takes the design values of the loop driver component. Then, it either sends stored result values directly back to the loop driver or it forwards the values to the actual evaluation loop. If the evaluation loop is done, the newly evaluated result values are fed back to the loop driver via the Evaluation Memory component, so that it can store the result values together with the design values sent before for later reuse. The values are stored in a file on the file system.

Usage

You need to configure two things: inputs/outputs and the path to the evaluation memory file wherein the values are stored respectively should be stored.

The inputs and outputs are configured in the "Inputs/Outputs" configuration tab. There are five tables whereby three of them are read-only. In the first one (seen from left to right and up to down), create an input for each design value sent from the evaluation loop driver to the loop. For each input an output is created as well in the second table. They are used in case the design values are just forwarded into the loop. In the third table, create one input for each result value evaluated by the loop. Again, for each input an output is created as well, this time it is created in the fourth table. They are used to forward the result values to the evaluation loop driver.

The path to the evaluation memory file is either configured in the 'Memory File' configuration tab or is defined at workflow start if the checkbox is checked appropriately.

Finally, the inputs and outputs must be connected to the evaluation loop driver and to the loop properly. If done, there are actually two loops: evaluation loop driver - Evaluation Memory component and Evaluation Memory component - evaluation loop. The fist one is driven by the evaluation loop driver, the second one is driven by the Evaluation Memory component. Thereby, only the evaluation loop driver is able to know when the second loop is done. Therefore, the evaluation loop driver must be connected to the Evaluation Memory component: Connect the output 'Outer loop done' of the evaluation loop driver to the input 'Loop done' of the Evaluation Memory component.

Now the Evaluation Memory component is ready to use.
