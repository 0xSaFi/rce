Design of Experiments Component Usage

Synopsis

With the DOE component a bounded solution space can be sampled.

Rationale

The DOE component sends values (floating-point numbers) to other components. The values are either generated on the base of a design method or are provided by a custom design table. The values can be used to sample a solution within a bounded space. They are independent to each other.

Usage

First, outputs must be defined. Each output is of type float and has a lower and an upper bound. The definition of inputs is optional. If inputs are defined, the DOE component maps one set of output values to one set of input values. I.e., output values are sent as soon as input values (corresponding to previously sent output values) are received. Queuing of input values as not allowed. If no inputs are defined, the output values are sent all at once at workflow start.


Second, the design method must be selected. You can choose between four methods:


Full factorial design
Latin hypercube design
Monte Carlo design
Custom design


The first three methods generate the output values on the base of established design methods (see links above). For two of them, the values are random. You can choose a seed in order to reuse the same values later. The number of samples can be defined with the option "Number of levels"/"Desired runs".
The last method allows to define a custom design table. The table at the bottom is editable and values can be entered. The table can be saved as a csv file and can be re-loaded later on. It is also possible to define a custom sample range by modifying the "Start at sample #" and "End at sample #" option.
