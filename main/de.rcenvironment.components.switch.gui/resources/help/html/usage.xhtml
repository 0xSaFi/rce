Switch Component Usage

Synopsis

The Switch component forwards an input value to one of two ouputs.

Rationale

The Switch component allows to direct the data flow within a workflow. It receives input values at one pre-defined input and forwards them to one of two pre-defined outputs. To which one the values are forwarded depends on a condition. The condition is configured by the user within the Switch component. It can be based on the values received at the pre-defined input or on values received at additional inputs.

Usage

The Switch component has a pre-defined input 'To forward'. Values, which are received at that input are forwarded to one of the two outputs. That input is a so-called 'data input'. The default data type of the data input is float. Change the data type to the one you require to forward. Next to that input, you can create as many inputs as you like. They are intended to be used within the condition definition and are so-called 'condition inputs'. Note: If you like to direct data of multiple inputs you need to add multiple Switch components to your workflow.

Define your condition within the 'Condition' tab. A condition may contain input names, numbers, relational operators, and parenthesis. You can either insert input names and valid operators from the drop down list on the right-hand side or you can write them by yourself. Inputs can only be used within conditions if their data types is either integer, float or boolean. The data input automatically appears in the drop down list, if it has one of the data types allowed.
